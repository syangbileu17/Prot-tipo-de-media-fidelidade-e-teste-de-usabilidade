# 📚 Projeto de Interface – Portal Acadêmico com Sistema de Solicitações

## 📝 Descrição do Projeto

Este projeto consiste no desenvolvimento de um protótipo de interface desktop criado no Figma para um **Portal Acadêmico de Autoatendimento**, conforme representado nas telas apresentadas.

A interface simulada na imagem demonstra um sistema utilizado por alunos para realizar solicitações acadêmicas, acompanhar seus pedidos e acessar informações importantes de forma rápida e organizada. O sistema foi estruturado com um menu lateral, telas de formulário, listagem de solicitações e acompanhamento de status.

O projeto está diretamente relacionado ao conceito de **Portal de Autoatendimento de Serviços**, adaptado para o contexto acadêmico, onde estudantes conseguem resolver demandas sem precisar de atendimento presencial.

---

## 🎯 Objetivo do Projeto

Desenvolver uma interface intuitiva e eficiente que permita ao usuário:

* Registrar solicitações acadêmicas com facilidade
* Acompanhar o andamento dos pedidos
* Visualizar informações detalhadas
* Navegar rapidamente entre funcionalidades

---

## 🖥️ Relação com a Interface Desenvolvida (Imagem)

Com base na imagem do Figma, o sistema apresenta as seguintes telas principais:

### 🔑 Tela de Login

* Acesso ao sistema por meio de autenticação
* Interface simples e direta, com foco em rapidez

### 📋 Área do Aluno (Menu Lateral)

* Navegação estruturada com opções como:

  * Início
  * Minhas Solicitações
  * Nova Solicitação
  * Histórico
* Organização que facilita o acesso às funcionalidades principais

### 📝 Tela de Nova Solicitação

* Formulário para abertura de chamados
* Campos como tipo de solicitação, descrição e prioridade
* Botão de confirmação com feedback visual

### 📊 Tela de Solicitações

* Listagem das solicitações realizadas
* Exibição de status (ex: em andamento, concluído)
* Visualização rápida das informações principais

### 📄 Tela de Detalhes

* Informações completas sobre uma solicitação
* Dados organizados para facilitar leitura e entendimento

---

## 👤 Perfil do Usuário

O sistema foi pensado para:

* Estudantes com pouca familiaridade técnica
* Usuários que precisam resolver demandas rapidamente
* Pessoas que buscam autonomia sem depender de suporte

---

## 🧠 Desafios de UX Trabalhados

O projeto buscou solucionar desafios importantes de experiência do usuário, como:

* Redução de complexidade em formulários
* Organização de múltiplas solicitações
* Clareza na exibição de status
* Feedback visual em ações realizadas
* Navegação simples e intuitiva

---

## 🎨 Design e Interface

A interface apresentada na imagem possui características como:

* Uso de cores suaves (predominância de tons de roxo)
* Layout limpo e organizado
* Componentes visuais consistentes (cards, botões e listas)
* Ícones ilustrativos para melhorar a compreensão
* Fluxo contínuo entre as telas
